# Симулятор RAM-машины. Машина с произвольным доступом к памяти

Веб-приложение для интерактивного моделирования машины с произвольным доступом к памяти (Random Access Machine) в учебных целях по дисциплинам «Теория автоматов» и «Алгоритмы». Приложение работает полностью на стороне клиента и не требует серверной части.

Функциональные возможности

Система команд
- Управление: HALT
- Ввод/вывод: READ, WRITE
- Операции с памятью: LOAD, STORE
- Арифметика: ADD, SUB, MULT, DIV, NEG
- Битовое сдвиги: LSHIFT, RSHIFT
- Переходы: JUMP, JZ, JG, JL

Предустановленные примеры
- Сложение двух чисел
- Умножение на константу
- Копирование в регистр
- Факториал
- Сумма N чисел
- Поиск максимума
- Возведение в степень

Режимы выполнения
- Пошаговый режим с подсветкой текущей команды
- Автоматический режим с настройкой задержки исполнения (50–2000 мс)
- Журнал выполнения (лог) с фиксацией этапов и времени

Пользовательский интерфейс
- Переключение тем: светлая по умолчанию, темная по выбору
- Адаптивная верстка
- Уведомления о состоянии выполнения (toast)
- Визуализация состояния: регистры, входная и выходная ленты, PC, текущая команда

Экспорт результатов
- Экспорт в CSV с кодировкой UTF-8
- Состав CSV: исходная программа (в одну строку), входные и выходные данные, количество шагов, время выполнения, финальные значения регистров, состояние PC, статус завершения
- Автоматическое именование файла с временной меткой

Технические требования
- Поддерживаемые браузеры: Chrome 90+, Firefox 88+, Safari 14+, Яндекс.Браузер 25+
- Минимальное разрешение: 1024×768
- Необходимые технологии: включенный JavaScript (ES6+), доступ к файловой системе браузера для скачивания CSV

Структура проекта
- index.html — интерфейс приложения
- styles.css — стили, в том числе оформление кнопки экспорта и переключение тем
- app.js — логика симулятора, обработка команд, парсер, управление режимами, экспорт CSV
- README.md — документация

Инструкция по запуску

Вариант 1: через локальный HTTP-сервер (рекомендуется)
1) Распаковать проект в каталог.
2) В терминале выполнить: python -m http.server 8000
3) Открыть в браузере: http://localhost:8000

Вариант 2: прямое открытие файла
1) Распаковать проект.
2) Открыть файл index.html двойным кликом.
3) При ограничениях безопасности рекомендуется использовать Firefox либо запуск через HTTP-сервер (см. Вариант 1).

Порядок работы
1) Выбрать один из предустановленных примеров либо ввести собственную программу.
2) Нажать «Загрузить программу», при необходимости указать входные данные.
3) Запустить выполнение (пошагово или автоматически).
4) По завершении, при необходимости выполнить экспорт результатов в CSV.

Формат языка программ
- Команды из перечня «Система команд»
- Операнды: немедленный (константа), прямая адресация [i], косвенная адресация [[i]], метки для переходов
- Допускаются комментарии в строке программы (если предусмотрено парсером)

Сообщения об ошибках и валидация
- Проверка синтаксиса команд и меток при загрузке
- Обработка ошибок выполнения (деление на ноль, выход за пределы входной ленты и т.п.)
- Информативные уведомления в интерфейсе с указанием причины и шага

Проверка установки
1) Запустить приложение.
2) Выбрать пример «Сложение двух чисел».
3) Загрузить программу и выполнить в автоматическом режиме.
4) Проверить вывод «8» в выходной ленте.
5) Выполнить экспорт CSV и открыть файл для проверки содержимого.

Назначение и область применения
- Учебные курсы по теории автоматов и алгоритмам
- Лабораторные работы и самостоятельная подготовка
- Демонстрации принципов вычислительных моделей RAM

Ограничения и допущения
- Объем визуализируемых регистров в интерфейсе ограничен для удобства; при обращении программой к новым индексам регистры добавляются динамически
- Производительность и скорость визуализации зависят от возможностей браузера и аппаратной платформы
